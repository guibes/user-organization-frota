seu primeiro MR
~ 2h

o primeiro merge request é simbólico — pode ser pequeno (typo na doc, um lint warning), o objetivo é exercitar o fluxo completo: branch → commit → pipeline → review → merge.

leia o guia de contribuição
conventional commits, padrão de PR, checklist de review.
abra seu primeiro MR (qualquer tamanho)
pode ser corrigir uma vírgula. o que importa é fechar o loop.
